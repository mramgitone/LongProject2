Creating Tag Groups

In your content in the XML Editor, you can group selected items within one of the following types of block tags: <blockquote>, <div>, <fieldset>, or <form>. Grouping selected items ensures that you can still work with the individual tags within that block tag as necessary.

    • blockquote This tag is typically used to format text used as a quotation. Usually the <blockquote> tag has margin indentations to set it apart from the rest of the content in the topic.

    • div A <div> tag is used to define logical divisions in your topics or to hold objects that can be "floated" (such as text boxes). It is similar to a paragraph tag, but it divides the page into larger sections. You can use <div> tags to name those larger sections of a topic and then affect them with stylesheets or Dynamic HTML. One common use of a <div> tag is to indent lots of content. For example, you might have a section of content containing four paragraphs, a numbered list, and an image. Rather than creating special style classes for all of those different elements with an indentation setting (e.g., margin-left or padding-left) on each, you can place your indentation setting on the <div> tag. That way, any content contained within that <div> tag will be indented accordingly.

    • fieldset This tag is used to combine multiple tags into a group, drawing a box around all of the content.

    • form This tag is used to create a form for user input. After you create the group with the <form> tag, you can open the topic in the Internal Text Editor to supply the necessary code for the form fields.

 note

If you are attempting to simply indent content, see Indenting Paragraphs or Indenting Lists. The reason the Tab key is not used for indenting content in is that it is not a best practice to format content locally in that way. Instead, it is recommended that you use styles to format content whenever possible.
