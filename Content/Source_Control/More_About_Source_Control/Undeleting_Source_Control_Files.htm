Undeleting Source Control Files

If you have a file marked as "Deleted" but then want to add it to source control, you can undelete the file, which then allows you to do this. You might encounter this situation if you rename a file and then create a new file with that name, or if you rename a file to the name of a previously deleted file.

how to undelete a source control file
  1. In one of the window panes (e.g., Content Explorer, Project Organizer, File List window pane, Pending Check-Ins window pane), select the relevant files(s).
    • Pending Check-Ins Window Pane From the Source Control ribbon or View menu, open the Pending Check-Ins window pane. Select the files, right-click, and select Source Control>Undelete.

    • Ribbon In the File section select Undelete.

    • Menu Select File>Source Control>Undelete.

    • Standard Toolbar Click the down arrow next to the Source Control button and select Undelete.

example

example

Let's say you open the Content Explorer and drag a file from one folder to another. Then you create a topic with the same name in the original folder. That new topic has a "Deleted" status and must be undeleted before it is added to source control.

In order to add the new file to source control, you need to remove the "Deleted" status. Therefore, you right-click on the file, select Source Control, and choose Undelete.
