How Did You Do That? Autonumbers

In this template, we used autonumbers for chapter titles, figures captions, and table captions.

chapter numbers

  1. Create a style class of the <p> tag and give it a name. We named ours "ChapterNumber" (p.ChapterNumber). See Creating New Style Classes.

    This is the style we used for the chapter autonumbers. For the accompanying chapter titles (or headings), we used the <h1> style.

    why not use just one style?

    In other situations, we might have used the <h1> style for both the chapter number and title, which is more efficient. But we didn't do that in this template, and here's why. We wanted to use a design on the opening chapter pages where the chapter number appeared sideways and the title appeared horizontally across the top.

    Because of this, we needed to use two styles instead of one. If both the chapter number and title appeared on the same line, we would have used just one style.

         why not use a class of the h1 style?

    Why did we create a class of the <p> tag and not the <h1> tag?

    We certainly could have done that. Either way, there are some modifications that you would need to make to the style. In the end, we chose to create a class of the <p> tag in order to keep all of our heading styles similar (i.e., used for heading text and not autonumbers).

    Enter this in the print medium in the stylesheet:

    p.ChapterNumber

    color: #3e3e3e;

    letter-spacing: -1px;

  3. Create a topic to hold the autonumber. We named ours "ChapterNumber." See Creating Topics.

    We placed our topic in a custom folder called "Print Only Topics" that we created in the Content Explorer. We created this subfolder to hold all of our topics that are intended only for print-based output. See Creating Subfolders.

  4. The "ChapterNumber" topic that you see in the template is a bit unusual for one reason. There is no text in it. Instead, we typed a single space on the only line, and we applied the p.ChapterNumber style class to that empty paragraph. See Applying Styles to Topic Content.

    We're not concerned about the lack of content in the topic, because in the output the autonumber (which consists of the word CHAPTER and a number) will be shown. As an alternative, we could have typed the word "CHAPTER" in that topic and created the autonumber format with the number only. Either way, it will work.

    You don't need to create any other topics for the multiple chapters that you have. You can use this one topic for all of them.

  5. Apply a condition tag (we applied one named "PDF") to the topic file so that it won't show up in your online output. See Applying Condition Tags to Content.

  6. Whenever you create print-based output, you need an "outline TOC," which is basically a Flare TOC file (FLTOC) with links to all of the topics you want to include in the output. We created four of these outline TOCs in the project:

    8. Expand the AutoNumber group and choose the settings shown in the following image:.

      The mc-auto-number-format property is the one that allow the style to automatically display the proper figure number, rather than forcing us to type it manually.

      Notice the CF prefix in the autonumber format. As you will see below, the table autonumber style also has a prefix labeled CT. This is necessary to keep the numbering sequences separate. Otherwise, you might end up with a chapter that has three images and three tables where the numbering goes like this: Figure 1-1, Figure 1-2, Table 1-3, Figure 1-4, Table 1-5, Table 1-6. That's not what you want. What you want is this: Figure 1-1, Figure 1-2, Table 1-1, Figure 1-3, Table 1-2, Table 1-3.

         get the css (advanced users)

    Enter this in the print medium in the stylesheet:

    p.Figure

    {

    color: #404040;

    font-style: italic;

    mc-auto-number-class: FigureNum;

    mc-auto-number-format: 'CF:Figure {chapnum}-{n+} ';

    margin-left: 10px;

    padding-bottom: 10px;

    }

    Notice the CF prefix in the autonumber format. As you will see below, the table autonumber style also has a prefix labeled CT. This is necessary to keep the numbering sequences separate. Otherwise, you might end up with a chapter that has three images and three tables where the numbering goes like this: Figure 1-1, Figure 1-2, Table 1-3, Figure 1-4, Table 1-5, Table 1-6. That's not what you want. What you want is this: Figure 1-1, Figure 1-2, Table 1-1, Figure 1-3, Table 1-2, Table 1-3.

  3. Notice in the previous step that the mc-auto-number-class property has "FigureNum" after it. That is a character-level (or "span") style that we created to control the look of the figure number (span.FigureNum). That's how we were able to make the figure number green, while the manually typed text following it looks different.

    why not use the table caption feature in the table properties?

    There is an option in the Table Properties dialog that lets you specify a caption for a table. And you can use the caption style to apply an autonumber to it. We could have done that instead of adding a special paragraph above the table. The main reason we chose to use the paragraph instead of the caption style has to do with rounded corners on the table. Different browsers treat captions differently on tables. Some browsers consider the caption part of the table, and others don't. So if you apply rounded borders to a table like we did, some browsers may put the caption within those rounded borders, and we didn't want that.
