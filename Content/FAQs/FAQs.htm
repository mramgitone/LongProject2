general how are topic titles, headings, and file names different and when are they used?

A title is the text that is found in the <title> tag of a topic's XHTML code, which you can see by opening a topic in the Internal Text Editor. You can set the title in the Properties dialog for a topic. See Changing Topic Titles.

By contrast, a heading is the text found in a topic's content, with one of the CSS heading styles applied to it (<h1> through <h6>).

A file name is the name you gave the topic in the Content Explorer.

 how can i prevent flare from trying to connect to the internet?

You can perform the following steps. Keep in mind that these steps may vary slightly for different operating systems. Also, keep in mind that if you disable this activity, certain features in Flare may be affected (e.g., using a destination file to publish output to a server).

  1. Click the Start menu, type windows firewall, and press Enter. The Windows Firewall with Advanced Security window opens.
  2. On the left, click Inbound Rules.
  3. On the right, click New Rule. The New Inbound Rule Wizard opens.
  4. Select Program.
  5. Click Browse and navigate to the location where Flare is installed. For example, it might be C:\Program Files\MadCap Software\MadCap Flare V\Flare.app. Please note that this is just an example. Your files might be installed in a different folder than C:\Program Files.
  6. Select Flare.exe and click Open.
  7. Click Next.
  8. Click Block the connection.
  9. Click Next.
  10. Select which domains you wish to block, or just leave all selected and click Next.
  11. In the Name field type MadCap Flare V.
  12. Click Finish.
  13. On the left, click Outbound Rules.
  14. Repeat steps 3-12.
 i moved some things around in the interface and cannot figure out how to move them back to the way they were. how can i return to the layout that i started with?

From the Window menu, select Window>Reset Layout (if using the ribbon view) or Layouts>Reset Window Layout (if using the menu view). You are asked if you want to reset the current window layout to the factory settings. Click Yes.

 what is the difference between resetting and reloading layouts?

Resetting a layout returns the interface to the way it was when you opened for the first time. Reloading a layout means returning the interface to your last saved layout.

 what are these blank check boxes that i see next to files and folders in the content explorer and other places?

They are not check boxes. They are actually boxes used to show whether something has a condition tag applied to it. Condition tags are used if you want to produce different outputs.

example

example

You might have one output where you want to include all content and files, but you might have another output where you want to exclude some content and files from the output. Condition tags are associated with colors when you create them, so when you apply a condition tag to a file or folder in the Content Explorer, the box next to that item will contain the color of the condition tag. If you apply more than one condition tag to an item, the box will show all of the colors.

For more information see About Condition Tags, Creating Condition Tags, and Applying Condition Tags to Content.

 how can i see links to a particular file? for example, if i want to see what topics or images are linked to a particular topic, how can i do this?

Here are a couple of ways to do this:

  • Select View>Link Viewer. The Link Viewer window pane opens by default on the right side of the workspace. Next, open a file (e.g., topic, stylesheet, table of contents). As long as that file has focus (e.g., click in it), the related links are shown in the Link Viewer window pane.

    OR

  • In one of the window panes containing files (e.g., Content Explorer, Project Organizer, File List window pane), right-click a file and from the context menu select View Links. If it's not already open, the Link Viewer window pane opens by default on the right side of the workspace, showing the related links.

For more information see Viewing File Links.

note

Whenever you switch focus to another file (i.e., open it and click on it), the links in the Link Viewer window pane change accordingly.

 i imported a robohelp project into flare. in my robohelp project, i had headers and footers in a topic template. does flare use templates like i'm used to using them in robohelp? where are my imported headers and footers in flare? are they in a template?

In , templates are used when you create a new project, as well as when you add new content (e.g., topics, snippets, stylesheets) or new project-related elements (e.g., tables of contents, targets, skins) to it. A template is simply an existing file or element of the same type that you are creating. By basing your addition on a template, you are giving yourself a head start because the template already contains some content, settings, or formatting for you.

If you want to use something like topic templates in Flare, you can create a master page and associate it with a target. Like a topic template in RoboHelp, a master page in Flare lets you create a header and footer for multiple topics. It also lets you automatically include breadcrumbs and a mini-TOC at the top and bottom of topics (which help users know where the topic is placed in relation to other topics in the table of contents).

If you import a RoboHelp project that has topic templates containing headers and footers, your headers and footers are saved as snippets in Flare (stored in the Resources\Snippets subfolder in Content Explorer).

 xml (content) editor when working in the xml editor, the look of my topic completely changed (e.g., different fonts). what happened and how do i fix it?

Most likely, this occurred because the XML Editor switched from using one style medium (e.g., default) to another (e.g., print).

When you switch from Web Layout mode to Print Layout mode, the medium automatically changes with it ("default" medium for Web Layout mode and "print" medium for Print Layout mode). This means that the editor will show the content in the style associated with that medium. Therefore, if you want to see a different medium (e.g., default, non-print) for that topic, you must select it from the Medium drop-down in the XML Editor toolbar. If you are unable to see the Medium drop-down field in the XML Editor toolbar, you may need to maximize the program window or click the drop-down arrow at the end of the toolbar.

For more information see About Stylesheet Mediums and About Layout Modes.

 how do i view the tags and markup in the xml editor?

In the local toolbar of the XML Editor, click the face of the Show tags button (not the down arrow). Please note, however, that this button only shows the tags in the topic. It is not the true markup code that you can edit.

If you want to view and modify the true code, including text, tags, and metadata, you can open the topic in the Internal Text Editor or another application such as Notepad. To do this, click the Text Editor tab at the bottom of the XML Editor, or click and drag the bottom bar up to work in split view (seeing both the XML Editor and the Internal Text Editor). You can also right-click on the file in the Content Explorer and select Open with>Internal Text Editor (or another editor). See Internal Text Editor.

 what are all these vertical and horizontal bars that i see around my topic in the xml editor?

The XML Editor provides bars above and to the left of the content area. These bars provide you with information about your content without having to view all of the tags mixed within the text. They also let you perform certain tasks for sections of content by right-clicking the block bar and selecting an action. See About Structure Bars.
