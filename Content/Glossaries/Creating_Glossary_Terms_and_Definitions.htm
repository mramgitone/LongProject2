Creating Glossary Terms and Definitions

After you add a new glossary or open an existing one, you can create terms and definitions. In online output, users can access the terms and definitions in the Glossary pane (or tab).

how to Create Glossary Terms and Definitions
  1. If you do not have one already, add a glossary file. See Adding a Glossary File.
  2. From the Project Organizer, open the glossary to which you want to add glossary entries.
  3. In the local toolbar of the Glossary Editor, click to add a new item. A new row is added to the glossary, with "MyTerm" shown as the default name. The Properties dialog for the glossary term opens.
  4. In the Terms section, replace the text "MyTerm," and type the new term.
  5. In the Definition section, select either Text (if you want to provide a simple text definition for the term) or Topic (if you have created a topic for the definition and want the term to link to it). For print-based output, it probably makes the most sense to use the "Text" option.
  6. If you selected "Text," type a definition in the space next to that option. If you selected "Topic," click , then find and select a topic.
  7. If you plan to use glossary term links (see Inserting Glossary Term Links), click the Style tab. Then select one of the styles (Expanding, Popup, Hyperlink), which dictates how the definition for the glossary term link is displayed in the output.
    • Expanding This displays the definition in expanding text when the link is clicked.
    • Popup This displays the definition (or linked topic) in a popup when the user hovers the cursor over the link.
    • Hyperlink This opens the linked topic (as specified in Step 5) in its own window, as opposed to a popup. If the definition is provided as simple text rather than a linked topic, this option opens that glossary term link as a popup.
    8. note

    By default, glossary terms will not be converted to links if they are found in <h1> through <h6> styles, as well as hyperlinks (i.e., content with the <a> tag). However, if the same term is found in, say, a regular paragraph, the term will be converted to a link. If you want to avoid certain terms being converted to links automatically (or if you want to reverse this setting for <h1>-<h6> and hyperlink styles), you can use an option to ignore glossary terms. See Ignoring Glossary Terms in Styles.

After you create glossary terms and definitions, you need to enable glossaries in the skin you want to use for the target. See Enabling Glossaries in Skins.

If you are creating print-based output and want the entire glossary to be included in the manual (in addition to any glossary term links that are converted to footnotes), you also need to create a topic and insert a glossary proxy into it. In the output, the proxy is replaced by the generated glossary. See Creating a Topic for a Print Glossary.
