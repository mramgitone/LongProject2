Conditions Example

Following is an example of how conditions might be used in Mimic.

example

example

Let's say that you are creating a movie that will be shown to two different audiences. Perhaps all of the frames in the movie are appropriate for each audience, except for the final five frames, which should be shown only to Audience A. In that case, you might create two condition tags—one called "AudienceA" and the other "AudienceB."

Then you can mark each of the final five frames so that they are associated with the AudienceA marker.

In addition, you can create two targets—perhaps one is called "OutputA" and the other "OutputB."

In the OutputA target, you tell Mimic to include the AudienceA condition tag and exclude the AudienceB condition tag. In the OutputB target, you do the opposite.

When you generate the movie outputs, the movie using OutputA will show the final five frames. The movie using OutputB will not show the final five frames.

Steps for Using Conditions

steps for using conditions

Following are the basic steps involved with condition tags.

  1. (Optional) Link to Flare Project Read more… If you want to use existing condition tags from a MadCap Flare project, you can link your Mimic collection or movie to that Flare project. There are two ways to do this. The easiest method is to insert the Mimic collection or movie into a Flare project. When you open that collection or movie in to edit it, the condition tags from the Flare project are already available. The second method is to manually link the Mimic collection or movie to the Flare project from within . For the first method, see the online Help in Flare. For the second method, see Linking to Flare Projects.

  2. (Optional) Create Condition Tags Read more… Whether or not you intend to use existing condition tags from a Flare project, you always have the option of creating your own condition tags from within . In addition, when you create a new Mimic collection or movie, two condition tags are already provided for you—Demo and Practice. You can use these condition tags, rename them, or delete them if you wish. See Creating Condition Tags.

  3. Apply Condition Tags Read more… See Applying Condition Tags.

    note

    After you record a movie, you might notice that some objects (i.e., cursors and invisible rectangles) already have condition tags applied. That is why you see small red and blue squares on those objects. The reason for this is that Mimic provides you with two condition tags to get started—Demo and Practice. By default the Demo condition tag is associated with blue, and the Practice condition tag is associated with red. If you wanted to produce demo output that would simply show actions, you would most likely want to include the cursors, but not the invisible rectangles, which end users can click to perform the actions. This is why cursors are already associated the Demo condition tag and the action rectangles are associated with the Practice condition tag. You can always remove these condition tags from any of the objects. And if you do not want to use either the Demo or the Practice condition tag, you can delete them; the conditions will therefore be automatically removed from the objects, although the objects themselves will still exist in the frames.
  4. Associate Condition Tags with Movies, Collections, or Targets Read more… After applying condition tags to the appropriate elements, you can tell Mimic to include or exclude those conditions in the output. This can be done in three places—at the movie level (if it is a standalone movie), at the collection level, or on individual targets. In most cases, you will probably associate condition tags with targets, which is recommended. See Associating Condition Tags.
