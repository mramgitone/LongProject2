Creating Alignment Projects

You can create alignment projects after launching MadCap Align. An alignment project lets you leverage translation work previously completed outside of , uploading translated segments to your translation memory database.

how to create an alignment project

    • Ribbon In the Launch section select MadCap Align.

    • Menu Select Tools>MadCap Align.

  2. In MadCap Align do one of the following, depending on the part of the user interface you are using.

    • Ribbon Select File>New Align Project.
    • Menu Select File>New Align Project.

    • Standard Toolbar Click .

    The Start Alignment Wizard opens.

    • Ribbon Select File>New Align Project.

    • Menu Select File>New Align Project.

    • Standard Toolbar Click .

    The Start Alignment Wizard opens.

  4. Next to the Original Version field, click .

  5. In the dialog that opens, navigate to the original project or file (in the original language) that you have already translated, and double-click the appropriate file.

    notes

    Some file types are not supported in alignment projects.
  6. Next to the Translated Version field, click .
  7. In the dialog that opens, navigate to the previously translated version of that same project or file and double-click it.

  8. (Optional) If you want to automatically upload previously translated segments to translation memory, click the check box. Your translation memory database should be displayed in the field below. If it is not, you can click in the field and select it.

    notes

    If you select this check box, any uploaded segments that are already matched 100% will not be displayed in the File Map window pane in MadCap Align. Only files with segments that require alignment will appear in that window pane.

    If you do not use this feature, you will need to manually upload the translated segments to translation memory after the alignment project is created.

    In order to use this feature, you must first install a translation memory database. See About Translation Memory and Installing a Translation Memory Database. See the Lingo online Help.
  9. Click in the next field and select a translation memory database.
  10. Click Next.
  11. In the Source language section, select the original language that is used in the project you are translating. In the Target language section, select the language that you want to use for the translation.
  12. Click Next.
  13. In the Project Name field, type an appropriate name for your project.
  14. By default, a path to the Documents\My Aligned Projects folder on your hard drive is entered in the Project Folder field. (This folder is created when you install the program.) All subfolders and files related to the aligned project will be placed in this folder as you work on the project. Continue with the next step, unless you want to have your project files placed in a different folder. If so, do the following.
    1. Click . The Select Folder dialog opens.
    2. Navigate to the folder you want, select it, and click OK.
    note
  15. Click Finish.

When you create an alignment project, a file with an .lialign extension is automatically generated (e.g., My Project Version 2 Italian.lialign). This is the main file for the alignment project and is stored at the root level of the project folder in Windows (e.g., Documents\My Aligned Projects\My Project Version 2 Italian\My Project Version 2 Italian.lialign). You can open the project by double-clicking this file. See Opening Projects.

Although translated segments can automatically be uploaded to a translation memory database during the process of creating an alignment project, there might be some files or segments that require manual alignment. In the alignment project, the original and translated files and segments are shown side by side, allowing you to make sure everything is correct. If any pair of files or segments do not match precisely, you can manually align them. Once the alignment process is completed, you can manually upload those translated files or segments to your translation memory database. See Aligning Files and Aligning Segments.
