Feedback Admin

The Feedback Admin serves as the main control panel for configuring a Feedback file, which acts as the connector between Feedback Explorer and the Flare target to be tracked. This editor is used to perform tasks such as enabling anonymous comments, requiring new comments to be reviewed, adding administrators, adding reviewers, and excluding IP addresses.

  • Administrators (add/remove) Read more… See Adding Administrators and Removing Administrators.

    note
  • Comments (enable/disable anonymous) Read more… See Enabling Anonymous Comments.
  • Comments (hide automatically) Read more… You can hide comments automatically (i.e., after a specified number of days). You might use the automatic method, for example, if you frequently make updates to your output and all comments associated with the previous version therefore become outdated. See Hiding Comments.
  • Comments (require review) Read more… See Specifying Whether New Comments Need to be Reviewed.
  • Data (clear) Read more… See Clearing the Feedback Database.
  • Help system name (add) Read more… You can specify the name of the online documentation or Help system that belongs to a specific Feedback key. This name will be included in notification emails sent to administrators, making it easier to discern which output the email pertains to. See Adding a Help System Name.
  • IP addresses (exclude) Read more… You can specify that certain IP addresses should be excluded from Feedback data. This might be a useful feature, for example, if you want to test your online output but do not want your topic views and searches to be included in the data collected by the Feedback server. See Excluding IP Addresses.
  • Published Output URL (for viewing live topics in browser) Read more… If you publish output to a web server using one of the WebHelp outputs, you can take advantage of a "Live View" button in the Topic View window pane. This button opens the live topic in a browser window so that you can view it just as users are seeing it. In order to use this feature, you need to enter a published output URL to the Feedback Admin. See Viewing Live Topics in a Browser.
