Build Targets in Background

The “Use Madbuild (Work while compiling output)” option has been changed to “Build targets in background”. Also, the “Generate log files with Madbuild” option has been removed. Log files will always be built when the “Build targets in background” option is set.

 

Madbuild is Flare's build engine commonly used when generating output from the command line.

Flare now has an option to use Madbuild when generating target output. All builds and publishing requests can now be executed in Madbuild allowing the user to continue to work in Flare while the build is running in the background.

File->Options->Build

“Use Madbuild (Work while compiling output)” – this option enables the feature.

“Generate log files with Madbuild” – Any target built with Madbuild from Flare will log the target output

Build Window

An option has been added to the “Project” ribbon to open the build window

The first option clears all completed builds from the build window

The second option will view the output of the selected target build

The third option will open the log file of the selected target build

Double clicking a row will open the output of the target for viewing if the build completed or will open the log file of the build if the build failed.

When a build fails the progress bars change from green to red.

Related to madbuild, a build warning has been added to alert the user that a file was modified while a build was running

3d Object Support

Allow U3D files to be dragged into a topic

When users build online outputs with embedded u3d models, it won’t work on Firefox or Chrome unless they select the Adobe Reader plug-in to be the default PDF reader in the browser. I think this is important to note in the documentation for U3D models.

For Chrome – go to chrome://plugins/ page. Adobe Reader plug-in has to be enabled and the Chrome PDF viewer disabled

For Firefox, the Adobe Acrobat plug-in has to be selected from Firefox | Options | Applications | Portable Document Format

xxxxxxxxxxxxxxxxxxx

Users can now insert 3D models into Flare for PDF and web outputs. Since PDF support is limited to the U3D (Universal 3D) file type, this will be the only file type supported in Flare for now. Models will be treated as another type of multimedia and will follow the same workflow and UI. The only exception is that instead of being wrapped in <object> tags in the XML, 3D models will instead be wrapped in <MadCap:model3D> tags. In the corresponding Advanced dialog page within the Multimedia dialog, users can specify how they wish the model to appear in the output. Attributes include Activation State (activate model on click, document open, or page view), 3D toolbar visibility (this toolbar is provided by Adobe and allows users to change various parameters of the model’s rendering), Rendering Mode (appearance of model’s shape), Rendering Lighting, and Rendering Background Color. Standard CSS style attributes (such as size, background image, border, etc.) will still apply.

In PDF outputs, the model will be displayed within the document as shown below. For web outputs, each model is displayed within an iframe with a reference to a PDF document that is created at compile time which displays the corresponding model. Note that Adobe Reader must be used in order to view the 3D models without error; using a browser’s built-in PDF viewer will not render the model accurately.

 Accept Contribution Wizard Enhancements

The Accept Contribution Wizard workflow for viewing conflicts has been consolidated into one dialog window. The user will go through each of the conflicts and choose whether to use project files that already exist or import contribution files into the project. The choice to use project files will be selected by default. [Note: UI has been updated to fit the ‘Contribution’ header. These screen shots were taken before the fix.]

The user has two options to view/compare the highlighted conflict in a new dialog window by: 1) clicking the first button (with Tool Tip: “Compare Project and Contribution Files”) under the list view or 2) double-clicking on the conflict. The user may only view/compare one conflict at a time. Upon comparing the different files, the user can also select the appropriate file to save to the project.

When ‘Contribution’ is selected, the ‘Rename Contribution’ and ‘Contribution Path’ columns are pre-populated with existing project file name and path. The user may choose to overwrite the file this way, or they can rename or select a new file path to create a copy of the contribution file.

For convenience, there are two buttons that will allow the user to select either all project files or all contribution files.

The second button selects all project files (with Tool Tip: “Use Project File for All”). The third button selected all contribution files (with Tool Tip: “Use Contribution Files for All”).

To change one or more of the contribution paths, the user can simply highlight the conflict(s) and click on the fourth button (with Tool Tip: “Change Contribution Path”). This button will open a window to browse the project directory. When a new path is selected, the ‘Contribution Path’ will be updated accordingly in the Accept Contribution Wizard.

Miscellaneous features:

1) Columns can be sorted. File types are sorted by type and then alphabetically by the file name.

2) Columns can readjust their size to fit all of its contents by double-clicking on the header.

 Augmented Reality

Users can now create augmented reality (AR) projects that can be opened with Metaio Creator, an AR tool for non-programmers. The generated project is simply a package of image files (called ‘trackables’ within Metaio Creator) on which AR scenarios can be built. To create an AR project, open the Export Project dialog and choose “Augmented Reality Project” under Output (as seen in the first image). There are four different ways to specify which image files to include in the project: (1) entire project – all images within the Flare project’s content folder will be added, (2) conditions – all images tagged under the chosen condition(s) will be added, (3) file tags – all images tagged under the chosen file tag(s) will be added, and (4) select files – all images manually selected will be added. Note that Metaio Creator only supports png, bmp, jpg, jpeg, and gif image formats.

To open the generated project (of file type *.arp), the user must have Metaio Creator installed. A limited demo of the software can be downloaded for free; however, the AR project is limited to two images/trackables. The second image below is a screenshot of the AR project opened in Metaio Creator, while the third image is a screenshot from a mobile device demonstrating an AR scenario in action using an AR browser.

 Font Properties

A lot of improvements have been made to the Font Dialog Box that gets displayed when you click the Font Properties button on the ribbon in the XML Editor.

As you can see, the elements of the dialog page have been rearranged to support a larger ListBox. We’ve also added different categories to the list such as Pinned Fonts, Recently Used Fonts etc. All of these sections are collapsible and expandable. Just click on the little arrow on the groups banner. (Collapsed Group seen below)

To pin a font, simply hover over or select an option, then click the little pin icon that shows up on the left side. It will automatically be added to your Pinned Fonts group. To unpin a font click on the pin next to any font in the Pinned Fonts group.

The Last major change is a new button called “Font Families” which is used to create, edit, delete and use font families.

The “Add”, “Edit” and “Delete” buttons are pretty self-explanatory, when you are ready to use a font family, click on it and click “OK”. The font will be added to the Defined Font Sets group. Given that font families are not too popular anymore, some functionality was sacrificed for ease of design. Therefor font families do not really hold too much global state in the XML Editor. For example if you make a paragraph one font family, then proceed to edit that family in the Font Family Manager, the paragraph will not update with the edit of the family.

 Search Filter Ordering

Users can now customize the ordering of the Search Filter Set for web outputs. Previous behavior was to display the filters in alphabetical order (which remains the default behavior). In the Search FilterSet Editor, the ‘Order’ column now represents the displayed order in the output. To manipulate the order of the filters, use the up/down arrows in the toolbar or context menu. A set’s customized ordering is monitored with the additional “Order” attribute attached to necessary SearchFilter nodes in the corresponding XML file.

In the example displayed below, the previous behavior order would be First, Last, Second. With customized ordering, we can now make the order First, Second, Last.

 Variables

In Flare 10, DateTime variables were added to allow users to customize the format of the displayed date/time in a project. The displayed date/time represented the project’s build date/time. Now users have more options to what date/time is actually displayed in the output. There are five options:

- Manual: Initially the date/time that the variable was created, can be updated within a topic to the current time (both a specific instance or all instances of the selected variable, see below)

- File Creation: The creation date/time of the file the variable is contained in. Useful if you're creating a new file based on a template.

- File Save: The last save date/time of the file the variable is contained in

- Project Save: The latest save date/time of all files in the project

- Project Build: The date/time of the project build (same as v10)

To choose the type of DateTime variable, use the dropdown within the Edit Format Dialog. To update a manual DateTime variable, right click on the variable and choose to either update that specific instance of the variable or all instances throughout the project.

 XML Editor

The following enhancements have been made to the XML Editor.

  • You can now shift-click on blocks to extend the current selection to that block.

  • Block bars will now highlight correctly when your selection includes multiple blocks. Previously only the first block in the selection would be highlighted. Now blocks that are fully within your selection have a darker blue background, while partially selected blocks that your selection starts or ends on have a lighter blue background:

  • In Flare 11 when your cursor is at the start of a paragraph with no text selected, pressing tab will increase the text-indent of that paragraph by 0.5 inches. All other selection types (e.g. place the cursor anywhere else in the paragraph, select some of the paragraph or multiple paragraphs) will continue to invoke the “create group” dialog.
