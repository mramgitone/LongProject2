About Language Support

Although 's user interface is displayed in English only, many other languages are supported. Therefore, you can "localize" (i.e., select other languages for) your movies and collections internally and externally.

ways to use languages in mimic

Following are some of the ways that you can take advantage of language support in .

  • Use MadCap Lingo Integration Read more… One of the easiest ways to translate a collection or movie is for a translator to open that collection or movie within MadCap Lingo, which is tightly integrated with . Because of this integration, there is no need to transfer localized files outside of the actual collection or movie, which helps prevent content and formatting corruption. In addition, translators can leverage all previous translations created in other tools by importing Translation Memory eXchange (TMX) files. For more information, please refer to the documentation provided with MadCap Lingo.

  • Select the Spell Check Language  Read more… You can select a specific language to be used for spell checking a MadCap Mimic collection or movie. See Selecting a Language and About Spell Check.
  • Localize Movie Text  Read more… If your computer system is set up to type in another language, you can add text to objects (such as callouts and buttons) in that language. You can do this within the interface. In addition, because 's movie (MIMOV) and collection (MIPRJ) files are XML-based, these files can be delivered to a translator in order to localize the text. See Adding Text.

  • Select an Interface Language for Browser-based Output  Read more… There are two pieces to the generated output from movies: the content and the user interface (e.g., navigation buttons and labels). If you are generating certain output formats, you can affect the movie user interface by doing the following.

    • Selecting a Language Read more… Select a specific language skin for displaying the user interface. See Selecting a Language.
    • Editing a Language Skin Read more… This lets you change how the skin looks, as well as adjust text for navigation buttons. Normally, you would edit style settings for a skin in the Skin Editor. However if you want to display the output user interface in a particular language, you can use the Language Skin Editor to edit the style settings (including text) instead. (The rest of the interface settings, such as which buttons to include and where to place the navigation toolbar, should still be specified in the Skin Editor.) When you generate browser-based output, first looks at the Skin Editor. If it does not find manually entered settings in that editor, it uses those from the Language Skin Editor. If it does not find settings in either editor, it uses the default skin settings. See Editing Language Skins.
    • Creating a User-defined Language Skin Read more… In other words, if you happen to know a language for which does not provide a language skin (e.g., Bulgarian), you can produce your own language skin. See Creating User-Defined Language Skins.
