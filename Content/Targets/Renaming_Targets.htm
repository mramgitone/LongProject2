Renaming Targets

It is usually helpful to rename the targets that you use to reflect the nature of your project (especially if you are using multiple targets with the same output type).

example

example

If you are producing a Help systemmanual for a software application called FictionSoftPro, you might want to rename the target "FictionSoftPro." If you are creating a beginner's version and an advanced version of your Help systemmanual, you might want to name one target "Beginner FictionSoftPro" and the other target "Advanced FictionSoftPro." That would be much easier than trying to distinguish between targets called "MyPDF" and "Copy of MyPDF."
